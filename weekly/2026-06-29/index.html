Enterprise

Microsoft: Forward-Deployed Engineering

▼

The Move: $2.5 billion investment in new AI implementation unit with 6,000 employees embedded directly with clients doing AI work, following "forward deployed engineering" model.

This signals a major shift: Microsoft isn't selling AI tools and walking away. It's putting thousands of people into customer organizations to handle the hard work of AI adoption — governance, integration, cultural change, and ramp patterns.

📈 6,000 forward-deployed engineers embedded with clients
💡 L&D Lesson: Enablement and governance can't happen remotely. Embed L&D practitioners directly into teams doing AI work, not in centralized training centers.
Infrastructure

Amazon Quick: No-Code Onboarding Agents

▼

The Tool: Amazon Quick is a fully managed agentic service for HR: create no-code AI agents that answer new-hire questions, track compliance across systems (HR, IT, payroll), and clear tickets automatically.

This is the onboarding trend for 2026: agentic AI handles the routine work (account creation, doc collection, training scheduling) freeing humans for judgment calls and relationship-building during ramp.

⏱️ Up to 50% faster time-to-productivity; significantly higher retention
💡 L&D Lesson: Automate the repetitive, deterministic parts of onboarding. Use AI agents to handle sequential tasks. Save human attention for mentoring and cultural integration.
Product

Duolingo + GitHub: Copilot in Production

▼

The Integration: GitHub Copilot + Codespaces embedded in Duolingo's 300+ developer workflow for 2+ years. Real-time code suggestions, automated routine tasks, unified development environment.

No special training program. Engineers onboard to the tool naturally, learn by doing, and benefit from force-multiplier effects from day one.

📊 25% faster dev speed (new repos); 10% for experienced staff; 67% faster code review turnaround
💡 L&D Lesson: Remove environment setup friction (Codespaces) so new engineers can contribute to real work immediately. AI tooling is the training program — don't add a separate course layer.

Signal + move: How Lemonade's engineering culture can lead on AI governance and enablement

🎯 Judgment at Scale

As AI handles code review, pair programming, and test writing, junior engineers risk losing the opportunity to develop critical judgment. Design onboarding around this explicitly: teach "when to trust AI" and "when to challenge it," not just "how to use it."

Move: Add a PR template field: "Why do you trust this suggestion?" Make judgment-building visible from day one.

🚀 Forward-Deployed Enablement

Microsoft's model (embed 6,000 enablement specialists with clients) works for internal scale too. Rather than centralized L&D pushing content, embed learning practitioners directly with squads doing AI-intensive work—pair programming, code review, incident response.

Move: Pilot embedded "AI enablement engineer" role in one squad. Measure ramp time, quality metrics, and retention.

⚖️ Governance as Hiring Signal

Lemonade can differentiate talent acquisition by being clear about AI governance philosophy from day one. "We expect you to challenge AI, not just trust it" signals maturity and attracts senior engineers who care about judgment-driven culture.

Move: Build AI governance questions into engineering interviews. Make it a hiring differentiator.

📊 Skills as Behavioral Signals

Instead of self-reported skills surveys, mine GitHub activity (code complexity, review quality, learning velocity) + internal mobility signals. This reveals who's genuinely upskilling and ready for stretch roles.

Move: Build a simple dashboard of GitHub signals for mobility conversations. Use it in career development discussions.
